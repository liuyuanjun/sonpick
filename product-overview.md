# 拾音 Sonpick - 产品概览

## 定位

可部署在 NAS 上的个人音乐下载与管理 Web 应用。基于 musicdl 搜索下载，支持 WebDAV 上传/浏览/播放，以及本地曲库管理。

## 主要功能

- 搜索下载：歌名/歌手搜索，多格式选择
- 导入下载：歌单文本批量下载
- 本地曲库：播放、转码 MP3、上传 WebDAV
- 播放器：收藏/歌单/艺术家/专辑/歌曲/最近播放、歌词、队列与播放模式
- 曲库扫描：本地目录 + WebDAV 递归扫描入库，支持排除规则
- WebDAV：连接配置、目录浏览、代理播放、上传策略
- 操作日志：下载/上传/删除/转码/扫描可查询
- 全局播放器 + 白天/黑夜主题
- 单用户密码登录（JWT）

## 当前设计要点（0.10.0-rc3）

- **搜索曲库比对**：下载页搜索结果与 `Song + SongFile` 批量比对，标注「曲库已存在 / 疑似已存在」；三级匹配（平台曲目 ID / 规范化 artist+title+album+时长 / artist+title 疑似），remix/live/伴奏等版本差异只判疑似；命中版本展示格式与实际大小（本地版本大小缺失时按需 stat，不做全目录扫描）
- **下载前版本选择**：命中曲库时弹窗决策——默认「保留两者，下载为新版本」（新版本并入同一逻辑 Song）、可选「下载完成后替换所选版本」（同目录临时文件 + `os.replace` 原子替换，失败不动旧文件）或取消；WebDAV remote-only 版本仅提示存在，不支持替换
- **移动端页面适配扩展**：在播放器/曲库/设置之外，下载、概览、操作日志与登录页完成 ≤768px 布局适配；桌面端布局保持不变。
- **移动端底部导航**：激活 Tab 使用主题色 + 加粗 + 浅色底，与非激活态明显区分（经 `useThemeVars()` 取色，勿再使用未全局注入的 `--n-*` 变量）。
- **曲库源入口统一**：独立 `/sources` 路由继续重定向到曲库；管理入口统一进入曲库（移动端可带 `manage=1` 打开来源抽屉）。
- **迷你播放器无曲目时收起**：无当前歌曲不展示迷你条，也不预留底部占位（播放器页同步回收高度，不留空白区域）；移动端顶部进度线跟随真实播放进度。
- **播放器进度跳转**：播放器页滑块与歌词行均可跳转播放位置，由全局音频实例统一执行并同步进度和歌词高亮；暂停时跳转不会自动开始播放。
- **任务中心移动端抽屉**：宽度随视口自适应，避免固定宽度导致横向溢出。
- **概览页重设计**：欢迎区/正在播放、KPI、最近播放、完整度、曲库源与最近动态聚合，作为日常入口首页。
- 曲库页与设置页完成移动端适配：曲库用来源选择条 + 底部抽屉管理来源，设置页顶部 segment 切换与刮削源卡片化

- **手机端适配（≤768px）**：全局导航改为底部 Tab 条（概览/播放器/下载/曲库/设置），迷你播放条收紧并带顶部细进度线，点按展开全屏「正在播放」浮层；播放器页分区改为顶部横向 Tab，列表全宽、单击行即播放、操作按钮折叠进「更多」菜单；播放队列为全屏面板；桌面端布局不变。
- **多媒体源**：逻辑歌曲（Song）与歌曲文件版本（SongFile）分层；同一歌曲可关联多个 local / webdav 曲源及 FLAC、MP3 版本，列表仅展示一首逻辑歌曲，播放按格式偏好、曲源优先级与可用状态自动回退。
- **曲库页**：整合原歌曲源与曲库，左侧曲库卡片管理（500px 固定宽），右侧歌曲/浏览模式切换；歌曲列表两行高密度展示（歌曲/格式 + 图标操作）；浏览模式支持删除文件/目录（本地与 WebDAV），删除后清理关联曲库记录并落操作日志
- **格式分目录**：设置页配置 MP3 存放目录（默认 `<存储>/MP3`）与无损存放目录（默认 `<存储>/LOSSLESS`）；下载按格式自动落盘到对应目录；本地曲库整理可选「按格式归档」纠正错放文件
- **下载**：搜索/批量均支持选源（默认 QQ 音乐）；各源并发搜索 + 单源 300s 硬超时 + 每源 10 条；下载后以 musicdl 实际返回的输出路径入库，避免来源目录推断失败
- **任务中心**：顶栏图标 + 右侧抽屉，WS 实时展示进行中任务进度（含取消），下方附最近任务；添加时间用于排队等待，真正开始时间用于任务耗时
- **曲库卡片**：连通状态、上次扫描、歌曲数、默认上传源，并保留测试/扫描/整理/刮削等操作
- **扫描弹窗**：播放器选源/全部扫描，不再跳转设置
- **上传**：自动上传→默认 WebDAV 源；手动上传可选源并检测目标冲突，存在同名文件时弹窗确认覆盖
- **曲库歌曲**：按所选曲库筛选歌曲，仅展示当前源内实际可播放的版本；本地源校验文件真实存在；格式列展示版本可用状态，悬浮显示实际路径，默认进入本地曲库
- **曲库浏览**：WebDAV 走代理列表，本地曲库提供只读根目录内文件/文件夹浏览
- **下载页**：搜索 + 导入 Tab；搜索 20 条/页
- **概览**：歌曲数、元信息完整度、源统计、任务概况
- **元数据**：扫描优先内嵌标签；路径仅作弱启发式，过滤 Favorite 等集合目录
- **目录约定**：新下载按 `艺术家/专辑/` 落盘；歌词/封面与音频同目录
- **目录规范**：`library_layout` 统一 `艺术家/专辑/歌名`、`cover.jpg`、`artist.jpg`、同名 `.lrc`
- **元数据管线**：`resolve_song_meta` 内嵌→侧车→DB→多源网络（MusicBrainz→musicdl）；下载页展示规范说明
- **整理脚本**：`scripts/reorganize_library.py` 可独立运行（默认根=脚本目录，`-r` 指定）；dry-run / `--apply`；可选 `--with-db` / `--enrich`
- **整理/刮削**：整理支持选目录/数量/`_failed`；缺专辑默认跳过；刮削可配置国内/海外源，自动链路按本地元数据→国内主源→海外兜底→Chromaprint/AcoustID 指纹深挖
- **刮削源设置**：设置页提供来源启用、自动参与、优先级、地区及单源测试；海外包含 iTunes、Deezer、MusicBrainz + Cover Art Archive；AcoustID Key 在系统设置中加密保存
- **播放器舞台**：页面贴边展示无外层边距；歌曲列表与右侧播放器通过明暗主题变量、半透明雾面背景、低对比柔光接缝衔接，接缝渐变跟随当前封面主色；播放队列支持浅色/暗色可读背景；滚动条采用细窄低对比主题化样式；叠层模式黑胶右上自适应放大，歌词偏左透明叠放且隐藏滚动条；封面模式放大唱片与封面占比；封面/叠层/歌词三个模式图标同时展示并高亮当前项，标签/刮削/队列使用图标按钮与悬浮提示
- **播放器刮削**：舞台「刮削」当前曲；列表「刮削本页」；单曲 enrich 默认异步，避免反代超时
- **网易封面兜底**：候选搜索缺失 `cover_url` 时以歌曲详情接口补齐专辑封面；候选评分函数独立导入兜底函数，避免候选接口 500
- **查询清洗**：刮削前去除 QQ songmid/序号/质量标签污染，提高华语源命中率
- **SmartCN 刮削**：并行网易/咪咕/QQ + 打分排序；写标签含歌词/封面
- **WebDAV 浏览**：从曲库页 WebDAV 卡片或右侧浏览模式进入（非侧栏独立入口）
- **下载与刮削**：下载搜索/任务支持选择全部或 QQ 音乐源；QQ 请求串行重试并反馈失败原因；播放器刮削可查看文件路径、手动修改检索关键词
- **整理关联文件**：移动音频时同步迁移同名歌词、同名封面与目录级专辑封面
- **曲库页面**：来源卡片、歌曲工具栏与内容面板分区更清晰；歌曲列表/浏览文件采用带图标的分段切换；歌曲操作统一尺寸，删除为红色描边图标按钮，并适配窄屏
- **本地曲库连通性**：测试通过 `os.access(..., R_OK | X_OK)` 校验目录读取与进入权限，避免调用 `Path.is_readable()`。
- **版本可观测**：前后端 `0.10.0-rc3`，API 头 `X-App-Version`。
- **文件版本职责**：`SongFile` 是 local/WebDAV 物理路径、格式及封面/歌词侧车的唯一真相源；`Song` 只保留逻辑歌曲和聚合封面/歌词缓存，不再保存物理路径。
- **扫描异步化**：曲库扫描改为异步任务（type=scan），经 TaskWorker 后台执行，曲库页原地显示进度条，任务中心同步可见，完成后全局 toast 提示。
- **任务 watchdog**：后台 60s 扫描僵尸 running 任务（>30min 无更新），检查 worker 线程存活，失联则标记失败并推终态；解决容器重启/OOM 后任务永远卡在「进行中」的问题。
- **扫描路径自愈**：扫描前标记失效 SongFile 为 unavailable；扫描命中同歌同格式新路径时合并旧版本行；聚合封面/歌词由当前解析出的 SongFile 侧车回填到 Song。
- **SQLite 迁移顺序**：启动依次执行建表、增量字段迁移、默认媒体源、SongFile 索引以及 Song/SongFile 路径责任迁移；历史 `Song` 物理路径先回填到 SongFile，随后从 `songs` 表删除。


## 技术栈

- 后端：FastAPI + SQLite + SQLAlchemy 2.0
- 前端：Vue 3 + Vite + Naive UI + Pinia
- 下载引擎：musicdl
- 部署：Docker + docker-compose
